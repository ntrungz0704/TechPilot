# CP-00.0 — Governance baseline

Đây là contract đề xuất để Human Project Owner review hệ thống governance vừa được soạn. Trạng thái **PLAN_REVIEW** không cấp quyền triển khai sản phẩm và không xác nhận những thay đổi source đã tồn tại trước nhiệm vụ.

## Objective

Thiết lập một nguồn trạng thái lưu trong Git, độc lập công cụ, giúp Human, vibe coder, Writer và Reviewer biết đúng scope, quyền hạn, lifecycle, bằng chứng và điểm dừng.

## Business outcome

Giảm việc AI tự suy đoán, scope creep, review sai commit, thao tác release không được phép và lệch kiến trúc MVC.

## Dependencies and entry conditions

- Human xác nhận danh tính Human Project Owner và GitHub handle.
- Human quyết định cách xử lý working tree đang có thay đổi source lớn.
- Contract này được review; mọi approval phải được ghi lại trong repository.

## In scope

- Governance documents, onboarding, architecture record, lifecycle, templates và tool adapter Codex.
- Python standard-library checks, GitHub governance workflow, PR template và CODEOWNERS placeholder.

## Out of scope

- Mọi thay đổi trong techpilot/.
- Hợp thức hóa hoặc review product diff hiện tại.
- Bật branch protection trên GitHub, commit, push, merge, deploy hoặc release.
- Thêm runtime/development dependency cho ứng dụng.

## Technical contract

- Machine-readable ACTIVE dùng YAML front matter.
- Script chạy bằng Python standard library trên Windows và Linux CI.
- CI từ chối product change khi không có checkpoint active được duyệt.

## MVC constraints

- Rule kiến trúc mô tả PHP MVC thực tế; không ép DDD, Clean, Hexagonal, Repository hoặc Service layer.
- Governance bootstrap không thay đổi controller, model, view, route, schema hoặc runtime dependency.

## Existing conventions to preserve

- `techpilot/public/` là web root chuẩn và `public/index.php` là front controller.
- Controller, Model và View tiếp tục theo custom plain-PHP MVC đã quan sát; không phát minh layer mới.

## Required tests and evidence

- Chạy đúng các lệnh trong front matter `required_tests` và ghi command, result, exit code.
- Evidence phải liệt kê toàn bộ governance files và chứng minh checkpoint không sửa `techpilot/**`.

## Acceptance criteria

1. Tất cả file governance bắt buộc tồn tại và internal link hợp lệ.
2. ACTIVE parse được, dùng lifecycle hợp lệ và không giả định Owner, Writer, Reviewer hoặc approval.
3. repo_doctor chặn local work khi không có active checkpoint hoặc working tree không rõ nguồn.
4. governance_check chấp nhận cấu trúc governance hợp lệ nhưng từ chối product change ngoài allowlist.
5. Workflow dùng checkout full history, Python, repo doctor và governance check; không cài runtime dependency.
6. CODEOWNERS không phát minh GitHub username.
7. Không file nào dưới techpilot/ bị sửa bởi checkpoint governance này.

## Exit conditions

- Human review xong baseline và ghi quyết định cụ thể.

## Stop conditions

- Nếu cần sửa source, thêm dependency, đổi architecture hoặc làm rõ approval cũ: dừng và mở checkpoint riêng.

## Known risks

- Working tree có product delta tồn tại trước bootstrap và chưa có canonical contract/review.
- Human Project Owner và GitHub handle vẫn chưa được xác định.
